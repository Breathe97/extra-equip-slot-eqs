# extra-equip-slot-eqs 额外的装备栏

> 这是一个为饥荒联机版打造的轻量扩展模组，让角色拥有额外的服装、护符、背包和腰包装备槽。

📦 Steam 创意工坊：[额外的装备栏](https://steamcommunity.com/sharedfiles/filedetails/?id=2946486855)

## ✨ 功能简介

- 🧢 **头饰栏** — 用于装备非护甲的头饰类物品（花环、牛帽、眼球伞等）
- 🧥 **服装栏** — 用于装备服装类物品（雨衣、背心、花衬衫等）
- 🔗 **腰包栏** — 用于装备腰包类物品（指南针）
- 📿 **护符栏** — 用于佩戴护符类物品（重生护符、建造护符等）
- 🎒 **背包栏** — 用于装备背包类物品（背包、猪皮包、坎普斯袋等）
- 🤖 **智能识别** — 未手动适配的物品会根据标签或属性自动分配到合适的槽位
- ⚙️ **可配置** — 可单独开启或关闭各个额外装备栏及自动识别功能
- 🛠️ **兼容修复** — 修复了寄居蟹隐士、重生护符、建造护符折扣、指南针 HUD、容器堆叠、复活背包重开等兼容性问题

## 📋 默认装备组合

```text
武器 + 护甲 + 头盔 + 头饰 + 服装 + 腰包（指南针）+ 护符 + 背包
```

## 🧭 使用方法

1. 安装模组后进入游戏，额外装备栏会自动生效。
2. 你可以在模组设置中控制各个额外槽位是否启用。
3. 对于未适配的物品，模组会优先尝试自动识别并分配到合适的位置。

## 🖼 贴图渲染逻辑

### 符号位分工

游戏角色身上有两个相关的符号位：

| 符号 | 用途 | 渲染内容 |
|------|------|----------|
| **`swap_body`** | 身体贴图 | 护甲、服装、护符 |
| **`swap_body_tall`** | 背包贴图 | 所有背包类物品（含光谱背包的变色粒子） |
| **`swap_hat`** | 头部贴图 | 头盔、帽子、头饰 |

### 动态符号收集

v2.4.0 重构了贴图渲染机制，不再硬编码物品的 build 和 symbol 映射，改为**运行时动态拦截**：

1. 劫持 `inventory:Equip()`，在装备物品时用一个代理对象替换 `owner.AnimState`
2. 物品的 `onequipfn` 执行时会调用 `OverrideSymbol` 或 `OverrideItemSkinSymbol`
3. 代理拦截这些调用，将 `{build, symbol}` 记录到 `items_anim_symbols` 表中（以 `prefab` 为 key）
4. 记录后放行，让原版调用正常执行
5. 监听 `equip`/`unequip` 事件，从 `items_anim_symbols` 查表并调用 `OverrideSymbol`/`OverrideItemSkinSymbol` 渲染

这样不再需要 `AMULET_SYMBOL_MAP` 硬编码，也不再需要为 `onemanband`、`sunkenchest`、`backcub` 等特殊物品做特判——它们的 `onequipfn` 自然会使用正确的 build 和 symbol，代理直接抓到即可。

### 执行顺序

```lua
-- 事件监听入口
RefreshEquipOverrides(_, data)
  → 判断事件来源 eslot

-- 头部渲染（is_head）
  → 重置发型/帽子显示状态
  → 优先取 HAT 槽，没有则取 HEAD 槽
  → render_item("swap_hat", item)  从 items_anim_symbols 查表渲染
  → 露顶帽/普通帽分别处理发型显示

-- 身体/背包渲染（is_body）
  → ClearOverrideSymbol("swap_body_tall")  清空旧背包
  → render_item("swap_body_tall", back_item)  背包贴图
  → render_item("swap_body", body_item)      护甲/服装/护符贴图
```

其中 `render_item` 统一查 `items_anim_symbols`，有皮肤走 `OverrideItemSkinSymbol`，无皮肤走 `OverrideSymbol`。

### 动态拦截的优势

- **无需硬编码映射表**——护符的 `torso_amulets`/`redamulet` 等符号从 `onequipfn` 自动捕获
- **自动兼容 mod 物品**——只要 mod 物品有正确的 `onequipfn`，无需额外适配
- **皮肤自动支持**——`OverrideItemSkinSymbol` 由代理自动转发
- **移除大量特判代码**——`onemanband`、`sunkenchest`、`backcub` 等特殊 case 全部删除

## 🛠 兼容性修复

| 修复项 | 说明 |
|--------|------|
| 🦀 **寄居蟹隐士** | 适配服装栏，使其能正常接收/拒绝外套类物品 |
| 💀 **重生护符** | 从 NECK 槽正确移除并销毁护符，清理残留贴图 |
| 🔨 **建造护符合成折扣** | 劫持 `GetEquippedItem` 回退查询 NECK → BODY |
| 🧭 **指南针 HUD** | 监听腰包栏的装备/卸下事件，正确开关指南针 HUD |
| 📦 **容器堆叠** | 重写 `GetOverflowContainer`，识别所有新增槽位的打开容器 |
| 👻 **鬼魂复活** | 大门/传送门复活后自动重新打开背包容器 |
| 🖥️ **服务端堆叠** | 服务端 `inventory:GetOverflowContainer()` 同样覆盖新增槽位 |

## ⚙️ 配置选项

| 选项 | 默认值 | 说明 |
|------|--------|------|
| 头饰栏 | 关闭 | 是否扩展头饰栏 |
| 服装栏 | 关闭 | 是否扩展服装栏 |
| 腰包栏 | 关闭 | 是否扩展腰包栏 |
| 护符栏 | 开启 | 是否扩展护符栏 |
| 背包栏 | 开启 | 是否扩展背包栏 |
| 自动识别服装 | 关闭 | 有防水/保暖属性的物品自动归入服装栏 |
| 自动识别护符 | 开启 | 有 `amulet` 标签的物品自动归入护符栏 |
| 自动识别背包 | 开启 | 有 `backpack`/`candybag` 标签的物品自动归入背包栏 |
